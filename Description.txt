1. VUE D'ENSEMBLE
-----------------
Nom du projet : SBI (Solution de Business Intelligente)
Version : 0.0.0
Type : Application web full-stack
Architecture : Multi-tenant (plusieurs clients/organisations sur une même instance)

Cette plateforme est une solution de Business Intelligence et d'analyse 
financière multi-sources. Elle permet aux utilisateurs de visualiser des 
reportings financiers, d'analyser les données comptables et métier, et 
d'interagir avec un assistant IA.

🏢 MULTI-TENANT :
   - Chaque client (tenant) dispose de son propre espace isolé
   - Données séparées et sécurisées entre les tenants
   - Configuration et personnalisation par tenant
   - Gestion centralisée des utilisateurs par organisation
   - Facturation et quotas par tenant (optionnel)

⚠️ ARCHITECTURE DÉCOUPLÉE :
   Cette application NE SE CONNECTE PAS directement aux bases de données sources.
   Les données sont collectées par des AGENTS DESKTOP séparés installés sur 
   les serveurs distants, qui communiquent avec cette plateforme via des APIs REST.

   Sources de données supportées :
   - Sage (Comptabilité, Gestion commerciale)
   - ERP tiers (SAP, Oracle, Dynamics, etc.)
   - Bases de données métier (SQL Server, PostgreSQL, MySQL, etc.)
   - Applications spécifiques (CRM, SIRH, etc.)

10. AGENT DESKTOP (Projet séparé à développer)
----------------------------------------------

L'Agent Desktop est une application Windows à développer séparément.
Elle sera installée sur chaque serveur hébergeant une base de données source.

Sources supportées:
   - Sage (Comptabilité, Gestion commerciale, Paie)
   - ERP tiers (SAP, Oracle, Microsoft Dynamics, Cegid, etc.)
   - Bases SQL Server, PostgreSQL, MySQL, Oracle
   - Applications métier avec accès base de données

Responsabilités:
   - Se connecter aux bases de données sources en LECTURE SEULE
   - Extraire les données selon les configurations définies
   - Pousser les données vers SBI via API REST
   - Gérer la planification des synchronisations
   - Envoyer des heartbeats pour le monitoring

Structure suggérée:
   Agent-BI/
   ├── src/
   │   ├── config/          # Configuration (connexions DB, URL API)
   │   ├── connectors/      # Connecteurs par type de base (SQL Server, PostgreSQL, etc.)
   │   ├── extractors/      # Extracteurs de données par source/table
   │   ├── sync/            # Logique de synchronisation
   │   ├── scheduler/       # Planification des tâches
   │   └── api_client/      # Client HTTP vers la plateforme
   ├── logs/                # Fichiers de logs
   ├── config.json          # Configuration utilisateur
   └── main.py / Program.cs # Point d'entrée

Fonctionnalités clés:
   ✓ Configuration multi-sources (plusieurs bases de données)
   ✓ Support des principaux SGBD (SQL Server, PostgreSQL, MySQL, Oracle)
   ✓ Sélection des sociétés/entités à synchroniser
   ✓ Mapping des tables/colonnes à extraire
   ✓ Planification flexible (toutes les X minutes/heures)
   ✓ Mode service Windows (démarrage automatique)
   ✓ Interface de configuration (optionnel)
   ✓ Gestion des erreurs et retry
   ✓ Compression des données envoyées
   ✓ Synchronisation incrémentale (delta)
